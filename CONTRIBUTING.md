# Contributing to Bayesian Stellar Parameters

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:
1. Check if the issue already exists in the [Issues](https://github.com/chomskyhonk/BayesianStellarParameters/issues)
2. If not, create a new issue with a clear description
3. Include relevant details: error messages, expected behavior, system info

### Contributing Code

1. **Fork the repository**
   ```bash
   git clone https://github.com/chomskyhonk/BayesianStellarParameters.git
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow the existing code style
   - Add comments where necessary
   - Update documentation if needed

4. **Test your changes**
   - Ensure existing functionality still works
   - Add tests for new features if applicable

5. **Commit your changes**
   ```bash
   git commit -am 'Add feature: description of feature'
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues

## Code Style

### C++
- Use C++11 or later standards
- Follow consistent indentation (4 spaces recommended)
- Use meaningful variable and function names
- Add comments for complex algorithms

### Python
- Follow PEP 8 style guidelines
- Use docstrings for functions and classes
- Keep functions focused and modular
- Add type hints where appropriate

## Documentation

- Update README files when adding new features
- Document new functions and classes
- Include examples for new functionality
- Keep the main README.md up to date

## Questions?

Feel free to open an issue for any questions about contributing!
